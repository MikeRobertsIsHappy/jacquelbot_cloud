Introduction

Many people enjoy being students of NVC and welcome opportunities to practice using their NVC skillset. An NVC skillset can be practiced at NVC classes, through NVC games like GROK, or occasionally through real life emotional issues.   Jacquelbot is a new way to practice NVC skills. Students are able to practice NVC skills by interacting with an online chatbot.  

Importantly, what separates Jacquelbot  from other NVC training tools is that Jacquelbot is capable of objectively scoring the student’s performance. Students can track their scores as they get better at using NVC.  Students will be able to interact with Jacquelbot in browser or through their favorite messaging app, like What’sApp or Facebook Messenger.

Jacquelbot is designed to play NVC games.

  • Modular - Because Jacquelbot is modular. It has the ability to grow quickly in different directions.
  • Persona models - Each training session (persona) will model a different emotional/need issue.
  • Scoring models - It will contain 4 scoring models.  Plus, it is imagined that additional scoring models can be added to help improve student performance.

Game sessions (called Personas) are easy to create.

  • Personas are defined in a spreadsheet.(The data model is below).   Each persona allows a very detailed conversation to be created.  It currently supports over 250 emotions and 60 needs and growing.  It allows an author to be very selective about the word and sentence usage for each stage of the conversation.
  • It is imagined that as the library of persona models grow, so will the library for basic conversations.  Basic libraries can be shared across many different personas.  Here are some examples of basic libraries that could  be created and shared.
  1. The current persona models do not contain responses for faux-feelings (abandoned, abused, rejected,...)   Once we have created a model for responding to faux-feelings, it can be used across many different personas.
  2. Models for stop words, unnecessary words, could be shared between personas.
  3. We could create a model to handle typical stupid questions to Jacquelbot, like “Do you like nvc?”,  Do you have feelings?” or things like that.

Every game is an opportunity to learn.  So, log everything. 

  • It is understood that the best feedback for improvement for Jacquelbot will be the errors from things it does not understand.   For example if someone types “I like pizza”, an irrelevant statement, we should be able to see from the logs what the response is and to correct that response in the future.  This is even more important when someone uses NVC in a novel way and Jacquelbot gives a poor response.   We should be able to take that learning and apply it to future sessions.  Jacquelbot has detailed logs that capture everything.  These logs can later be mined for data.
  • We can also use the data in the logs to improve the scoring.  For example, we can ask an NVC professional to interact with Jaquealbot, with all interactions being saved to the logs. Then we can use the logs to model that conversation as being a winning score for future students.    Students who have conversations similar to the NVC professional will receive a passing score.
  • Consider what type of data you would want after Jacquelbot is released on the internet and we have 100K conversations to evaluate?  Identifying patterns in big data sets, will enable the creation of Jacquelbot responses that can handle most situations.  
  1. Most common guessed feelings

  3. NVC scores for men over 30
  4. ...

NVC is a skillset and a mindset.  Games practice the skill sets.

NVC is a skillset and a mindset.  

  • The ‘skillset’ is both
  • a set of communication patterns/tools
  • vocabulary.
  • The ‘mindset’ is both
  • a desire for connection and
  • a stable emotional base that provides the foundation of the conversation.

Jacquelbot is designed to be a tool to help develop skillsets. (It is not clear how Jacquelbot can support the development of the correct mindset.)  

For the mindset, it is not expected for NVC students to try to fake the feeling of connection with the Jacquelbot personas.  However, we do hope that NVC students approach Jacquelbot with curiosity appropriate to any new game.

Realistic expectations of scoring.

  • To create a score for the conversation, a student’s responses will be compared against examples.  There are common NVC’ish ways to say things and those patterns can be cataloged for scoring.  However there will always be new ways to say something in NVC, that will receive a low score because it does not fit the current models well.  Developing a large catalog of responses will help mitigate the problem that new responses will be scored poorly.
  • Cultural issues will need to be addressed as Jacquelbot is released to different audiences. Even within the United States there are cultural differences, that create communication differences, that might require changes to the persona communication models. For example “y'all’’ (southern)  vs. ”you guys” (northern).  Regional specific models are expected.
  • Currently Jacquelbot does not compute a final score, but all the data described above is saved to the logs. There is not a solution for creating a final score yet.

Details

Below is a flow chart of the general process.  The first step in this model is for the NVC student (User) to check in and to try to find out what is alive in them.  Then the NVC student will give a response and it will be evaluated.

•Response models -There is not one model that can handle the complexity of the NVC communication. Several models are needed to be layered together to form a cohesive conversation.  For example, when determining the Jaquelbot’s response to the user input, several model-steps are used.

•Step 1 - Clean the user input of unnecessary words, also called  ‘stop words’.  Sentence analysis models often do better if some common stop words are removed like “the” and ”of”.  The cleaned sentence is easier to decode the meaning.

•Step 2 - Check if student_input is like “nvc help” or nvc_feelings”,  then form an appropriate response. This can print out a list of feelings or let the student exit the game

•Step 3 - Check if student_input  contains the correct feeling or sentences, based on word frequency analysis. This calculates the similarity between the student input and ‘example_inputs’’ by the number of words that they have in common.  We return a response appropriate to the example input with the greatest similarity to the student input..

•Step 4 – There are two ways that it can be understood

•We can develop training sets for common responses, such as guessing the wrong feeling/need. These responses would be authored inthe persona file.

•If user’s response does not contain a feeling/need but appears to be a well-formed sentence, we can send it to a separate general conversation bot to form an appropriate response. This bot is designed to respond to weird things and to direct the conversation back on target. This bot does not need to be part of the persona file as it can be reused for all personas.

•Step 5 - Response not understood-  If the response if not understood, Jacquelbot will try to ask the user to clarify

Scoring happens after the response is formed.

Each conversational unit has a specific communication goal.

The “goal of a conversational unit” can be defined several ways in a persona. The goal can be defined by either  response based or score based methods  

Response based goals, for example winning responses might be:

  • a winning response could be a correctly using a specific word (such as, “happy”)
  • Or using a specific phrase (such as, “you feeling sad”)  
  • Or using a phrase that is semantically similar to an example phrase
  • ...

Score based goals, for example winning responses might be:

  • A certain amount of total points for guessing the correct feelings (positive for good guesses, and negative for bad guesses)
  • Compute the mathematical similarity between two sentences reference
  • After 3 tries just move on to the next thing anyway.
  • ...

When the conversational units are linked together to create a cohesive personality to train with, it is called a persona.  A persona is defined by a file that has the information necessary to have the complete conversation across several conversational units.

1 unit persona

Enables simple games, like “guess the feeling”, based on certain prompt.

2 unit persona

Enables more complex games that guess the feeling and need with 2 different prompts

4 unit pesona

Enables more complex conversations that have several parts

While the initial version of Jacquelbot is designed for the 4 unit persona, it can be customized for other games, if desired.

How to score NVC points in the game.

Each sentence you type will be analyzed several different ways

  1. {'input_response_score': 100} How closely did you identify the correct feeling or need. For example  “feeling angry” or “need for peace.” Each practice persona will have 1 or more correct answers for each feeling/need. The other 300+ feeling/need options will be put into small groups based based on their similarity to the correct answer.  For example, If the correct feeling was “sad”, if you guessed the feeling words like “dismayed” or “worried”, you would score more points than if you guessed  words like “happy” or “joyful”.
  1. Could give negative scores for bad guesses
  2. Could give negative score for too many guesses
  1. {'did_it_sound_like_nvc': (0.58119, 0.41881)} Did the sentence sound like NVC? (likely lositive, likely negative) For example,  “are you feeling…” or “It seems like your need for …”  This is based on a large dictionary of the usual ways of saying things in NVC. This dictionary grows over time.  This is based on a trained chatbot analysis
  2. {'input_polarity': 0.45}Polarity measures emotional intensity.  Negative emotion<-> rational<->positive emotion.  Your responses should not be too strongly negative or positive, but it can depend on the correct feeling/need.
  3. {'input_subjectivity': 1.0} Subjectivity measures if the sentence declares a fact or declares an opinion. Generally, your sentences should tend to be more of a declaration of opinion, rather than a strong statement of fact.   NLTK out of the box.
  4. ...more scoring methods to be created

Two kinds of scores are collected for each input

  • User_input_response_scores - this is the scores on the raw input from the user
  • Cleaned_user_input_response_scores - this is the scores for the cleaned input from the user. Often times before a statement is analyzed by a machine the input will be cleaned and transformed to make it easier to understand for the computer. Examples of transformations might be fixing spelling mistakes, removing unnecessary white space, converting word to their root form, removing common words or duplicate words,...

It was important to capture the raw and the cleaned input so we can later evaluate cleaning methods

Each response is evaluated by 4 metrics and stored in the logs.

{'input_response_score': 100}  

{'did_it_sound_like_nvc': (0.58119, 0.41881)},

{'input_polarity': 0.45},

{'input_subjectivity': 1.0},

Totals are calculated in the app, but totals are not saved to the logs. It is reasoned that totals can be calculated later if needed from the data in the logs, so it does not need to be stored

This section details how to use the software.

A persona is defined in a Google Sheet.  

In this person file each conversational unit is it’s own worksheet in the spreadsheet

           

The tab ‘formatted_json’ collects the inputs from all the pages and presents them as valid json. Copying the content of column ‘K’, paste into a text document, save as json.

The ‘feeling’ conversational unit would have the standard set of dictionary headings

  • xxxx_conversation_data_examples - This is the main listing of conversation. In the present implementation it is set up with key words and responses. As, ["affectionate", "That is not quite the feeling”]   The goal of this data set is to make sure the response is aligned with the inputs.
  • xxxx_conversations_stop_words - This is used to remove common words from sentences to help improve accuracy on the conversation_data_examples analysis.
  • xxxx_sentence_analysis_data_examples - These sentences are  different and are for checking sentiment.  The goal of this data set is to check if the sentence was shaped like NVC.
  • xxxx_sentence_analysis_stop_words - These are stop words specific to sentiment analysis
  • xxxx_list -  a list of things (feelsing, needs, …) to present to the user on command
  • xxxx__clue - Not developed yet.. If the user wants a clue, they can get one.

Why Jaquelbot

I want to practice my NVC skills.  Practicing with people is very productive, but It can be hard sometimes to practice with others because of the need to schedule focused time for the practice.  Further, I had a need for flexibility and convenience.  I wanted that ability to have short practice sessions, on short notice.  And, I want to be able to pause in the middle of a training session, like I do when I practice with my French language phone app.  

With this background, Jacquelbot got started  when I was looking at my list of new year's resolutions. One of my resolutions was to create something in 2021: it was going to be either (1) a book on business in the silicon valley or (2) an NVC chatbot.   I was torn between which one to do. Long term vision of Jackalbot

The ultimate goal is for building a communication model(s) for evaluating NVC communication that can be reused in different ways in the future:

  • Can be used to gamify existing NVC games, like WeGROK
  • It can be used in robots that mimic facial expressions for training
  • It can be adapted to voice input rather than text input, in order to practice speaking NVC
  • It can be used in applications for video or robot based training.

Business applications of Jaquealbot

Because Jacquelbot can capture objective measurements on a person’s NVC communication skill sets, this tool might be valuable to corporations that would like to improve their employee’s communication skills and Emotional Quotient (EQ).  Jacquelbot provides the corporation both

(a) a training tool for employees to practice communication skills

(b) an evaluation tool to check employee learning.

A use case might be a company that would like to up-skill their customer service (hot line) team’s communication patterns.  Customized personas can be created for particular customer service issues, for interactions with both customers and team mates.

Corporations will have an easy time providing training to employees, since JAcquelbot can be accessed through a browser or through their favorite messaging app, like What’sApp or Facebook Messenger.
